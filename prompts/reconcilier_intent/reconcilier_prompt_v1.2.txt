## OPERATION TYPES (STRICT):
- SPLIT_AND_MODIFY: Extract 1 or more units from an existing item to create a new modified item.
- REPLACE_DISH: Exchange one dish type for another entirely.
- MODIFY_COMPONENT: Change internal parts (protein, sides, size) of an existing item.
- UPDATE_QUANTITY: Change the number of units.
- ADD_MODIFIER: Add/Remove specific ingredients in 'special_instructions'.
- UPDATE_PROTEIN_GLOBAL: Change protein across all applicable items in the order.

## RECOGNITION PATTERNS (Colombian Context):
- "más bien" / "mejor" -> Indicates a REPLACE or MODIFY operation.
- "uno de los..." -> Trigger SPLIT_AND_MODIFY.
- "en vez de A, B" -> Trigger MODIFY_COMPONENT.
- "regáleme con..." -> Indicates protein/side selection.

## BUSINESS RULES:
1. Total quantity must never be negative.
2. Inheritance: Preserve existing special instructions unless explicitly changed.
3. Default Size: If not mentioned, assume "corriente".
4. Instruction Conflict: If "sin X" is requested, remove any existing "con X".
